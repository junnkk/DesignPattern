## 싱글턴 패턴   
#### : 클래스 인스턴스를 하나만 만들고, 그 인스턴스로 전역 접근을 제공한다.   
   

> ##### 객체 지향 기초
- 추상화
- 캡슐화
- 다형성
- 상속


> ##### 객체 지향 원칙
- 바뀌는 부분은 캡슐화한다. 
- 상속보다는 구성을 활용한다. 
- 구현보다는 인터페이스에 맞춰서 프로그래밍한다. 
- 상호작용하는 객체 사이에는 가능하면 느슨한 결합을 사용해야 한다.
- 클래스는 확장에는 열려있어야 하지만 변경에는 닫혀 있어야 한다. 
- 추상화된 것에 의존하게 만들고 구상 클래스에 의존하지 않게 만든다.
